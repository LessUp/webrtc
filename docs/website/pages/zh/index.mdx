## 开发模型

这个仓库采用偏"收尾治理型"的 **OpenSpec 工作流**：

1. 先在 `openspec/` 中定义或更新行为
2. 按 change 里的任务实现
3. 做聚焦校验
4. 合并前完成 review

这样可以在保持项目体量克制的同时，让代码、文档和公开呈现始终一致。
